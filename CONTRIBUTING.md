# Contributing to Awesome Open Source SaaS

Thank you for your interest in contributing to this curated list of open-source SaaS projects!

## Guidelines for Adding a Project

Before submitting a pull request, please ensure your project meets the following criteria:

### Eligibility Requirements

1. **Open Source License** - The project must have a recognized open-source license (MIT, Apache 2.0, GPL, BSD, etc.)
2. **Actively Maintained** - The project should have commits within the last 6 months
3. **Production Ready** - The project should be stable enough for production use
4. **Well Documented** - Clear documentation, README, and setup instructions
5. **Self-Hostable** - The project should be suitable for self-hosting or local development
6. **Minimum Stars** - Projects should have at least 500 GitHub stars (exceptions may be made for exceptional projects)

### What We're Looking For

- Projects that provide viable alternatives to commercial SaaS products
- Tools that solve real problems for developers and businesses
- Projects with active communities and regular updates
- Software that follows security best practices

### What We're NOT Looking For

- Abandoned or unmaintained projects
- Projects without proper documentation
- Forks without significant improvements
- Projects with restrictive licenses that limit self-hosting
- Duplicate entries of already-listed projects

## How to Submit

### Adding a New Project

1. Fork this repository
2. Add your project to the appropriate category in `README.md`
3. Follow this format:

```markdown
### [Project Name](https://github.com/owner/repo)
![Stars](https://img.shields.io/github/stars/owner/repo?style=social)

<img src="https://avatars.githubusercontent.com/u/OWNER_ID?v=4" width="50" height="50" alt="Project Name logo">

A brief description of what the project does and why it's useful. Keep it to 1-2 sentences.
```

4. Ensure the project is placed in alphabetical order within its category
5. Submit a pull request with a clear title and description

### Suggesting a New Category

If you believe a new category is needed:

1. Open an issue first to discuss the new category
2. Provide at least 3 projects that would fit the new category
3. Explain why existing categories don't fit

### Reporting Issues

- **Broken Links** - Open an issue with the project name and broken link
- **Outdated Information** - Open an issue or submit a PR with corrections
- **Project No Longer Maintained** - Open an issue to discuss removal

## Pull Request Process

1. Ensure your PR follows the format guidelines above
2. Include a brief explanation of why the project should be added
3. One project per pull request (makes review easier)
4. Update any relevant sections (table of contents is auto-generated)

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.

## Questions?

If you have questions about whether a project fits, open an issue and we'll discuss it!
